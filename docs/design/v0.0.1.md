# sigflare

## 概述

一个最小化、可自托管的页面访问统计服务，基于 Cloudflare Worker 原生 `fetch` 接口接收上报请求，并将原始事件落到 ClickHouse。首个版本仅支持 `pv` 事件。

## 架构目标

- 极简实现：最少代码路径、最少默认值。
- 全链路保持原始：请求和响应直接使用原生 HTTP 对象。
- 数据层按 event 驱动：当前仅持久化 `pv`。
- 打包为 JS 后发布到 NPM，供其他项目复用。
- 支持在 Cloudflare 上部署，避免重框架封装。

## 技术栈

- TypeScript 作为开发语言
- Vite 作为打包工具（输出 JavaScript）
- Cloudflare Worker 原生 `Request`/`Response`
- ClickHouse HTTP 接口入库
- pnpm 作为包管理器

## API 接口

- GET `/sigflare.js`
  - 作用：返回可直接引入的统计脚本。
  - 成功响应：
    - 状态码 `200`
    - `content-type: application/javascript; charset=utf-8`
    - 行为：脚本在浏览器执行时发送一次 `event: "pv"` 到同域 `/collect`。

- POST `/collect`
  - 作用：接收埋点事件并写入 ClickHouse。
  - 入参要求：Content-Type 为 `application/json`，请求体为 JSON 字符串。
  - 必要字段：`event`，目前仅允许 `pv`。
  - 建议字段：`url`、`title`、`referrer`、`created_at`
  - 成功响应：`{ "ok": true, "event": "pv" }`
  - 错误响应示例：
    - `400` 非法参数（`event` 不是 `pv` 或 JSON 解析失败）
    - `405` 非 `POST`
    - `404` 路径不匹配
    - `502` ClickHouse 写入失败

- 数据落库
  - 目标字段：`event_type`, `received_at`, `request_method`, `request_path`, `request_url`, `request_headers`, `request_query`, `request_body`, `event_payload`
  - `request_headers` 原样保留原始 Header 键值对序列化
  - `request_body` 与 `event_payload` 保留原始 JSON 文本
  - 使用 ClickHouse 的 `FORMAT JSONEachRow` 插入

- 环境变量
  - `CLICKHOUSE_URL`
  - `CLICKHOUSE_DATABASE`
  - `CLICKHOUSE_TABLE`
  - `CLICKHOUSE_USER`（可选）
  - `CLICKHOUSE_PASSWORD`（可选）
